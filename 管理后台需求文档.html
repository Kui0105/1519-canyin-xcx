文档说明

本文档基于《万户山生态科技有限公司 - 项目需求文档 V2.0》第8节"后台管理系统需求"进行深化，结合当前移动端原型（v2.20.1）中已实现的全部用户端功能，逐项拆解为管理后台的细化功能需求、页面结构、数据字段、交互规则和接口需求。文档采用「一个模块 → 一个管理页面」的映射方式，确保前后端开发人员可直接参照本文档进行设计开发。

1
系统概述

管理后台是面向平台运营人员的PC端Web管理系统，用于对微信小程序（C端）的全部业务进行集中管理、监控和配置。采用Vue3 + Element Plus或React + Ant Design开发，与小程序共享后端API。

1.1 管理员角色与权限
角色优先级权限范围
超级管理员P0全部权限：系统配置、管理员账号管理、价格修改、功能开关、财务数据
内容审核员P0内容审核（通过/驳回/下架）、标签库、举报处理；不可修改价格与查看财务
运营编辑P1资讯发布编辑、Banner管理、热搜词配置、用户反馈处理
财务人员P1订单查看、对账导出、退款处理（需超级管理员审批）
1.2 全局框架布局

左侧导航菜单（常驻）+ 右侧内容区。一级菜单为功能模块，点击展开二级子菜单。

全局交互规则
  • 列表页默认分页20条，支持页码跳转和每页条数切换（20/50/100）
  • 所有列表支持导出 Excel（部分含敏感数据需权限）
  • 所有删除/封禁等不可逆操作需二次确认弹窗
  • 所有管理员操作写入操作日志，包含操作人、时间、IP、操作内容
  • 右上角显示当前管理员头像+名称+退出登录

2.3 封禁/解封
项目规则
封禁方式临时封禁（7/14/30天）+ 永久封禁
封禁原因（必填）下拉选择：发布违规信息 / 恶意骚扰 / 虚假身份 / 其他（手动补充）
封禁效果不可发布、不可付费、不可评论；已发布信息全部自动下架
解封效果恢复发布和付费权限；已下架信息不自动恢复，需手动操作

功能规则

余额调整管理员手动增减（退款到余额/活动赠送等），超管审批
余额流水查询按用户、日期、类型（充值/消费/退款）筛选；字段：用户信息、金额、余额、关联订单号、时间

审核操作
操作规则
审核通过信息立即上架，C端可见。若含置顶推广，按付费周期开始计算置顶时间
驳回弹出驳回原因对话框（必填）。原因内存预设：信息不实 / 图片不合规 / 分类错误 / 含有违规内容 / 联系方式无效 / 其他
批量操作支持勾选多条信息，批量通过或批量驳回（同一驳回原因）
驳回后C端表现
用户在"我的发布"中看到状态为"已驳回"，显示驳回原因，可点击编辑修改后重新提交。重新提交后进入新审核流程。
3.2 内容管理（已通过列表）
操作规则
查看详情查看信息完整内容及发布者信息
下架将已上架信息下架（需填写下架原因）。下架后C端不可见，用户"我的发布"中显示"已下架"状态
重新上架对已下架信息恢复上架（如误操作或用户申诉通过）
强制删除对严重违规信息进行物理删除（不可恢复），需超管权限+二次确认
3.3 敏感词管理 P2
功能规则
敏感词库列表展示所有敏感词；支持增删改；支持分类（政治/色情/暴力/广告/竞品/自定义）
审核高亮审核详情中自动检测并高亮命中敏感词
发布拦截C端用户发布时若文本命中敏感词，前端给出提示并阻止提交
3.4 举报管理
功能规则
举报列表字段：举报ID、被举报信息ID/标题、举报人、举报原因、举报时间、处理状态
处理操作【查看被举报内容】【忽略举报】【确认违规→下架信息】【封禁发布者】
处理结果通知处理完成后系统自动给举报人发送通知
4
内容运营
优先级：P0 核心 | 角色：超管、审核员、运营编辑
4.1 置顶管理
功能规则
置顶列表展示所有购买置顶的信息：信息ID/标题、发布者、置顶周期（7/15/30天）、开始时间、到期时间、剩余天数、状态（生效中/已到期）
置顶排序C端多信息置顶时按发布时间的先后顺序排列：最先发布的那条排最前
取消置顶管理员可手动取消置顶（需填原因），取消后信息恢复普通排序，已付费用不予退还
到期处理系统定时任务每天0点扫描到期置顶，自动将状态改为"已到期"，信息恢复普通排序
置顶续费用户在C端"我的发布"中可对置顶信息续费叠加周期，后台同步更新到期时间
4.2 标签库管理

对C端发布页的帖子标签（tag-chip）进行增删改管理。

功能规则
标签列表展示当前全部可选标签：标签ID、标签名、关联分类（全部分类通用 / 指定分类）、启用状态、排序权重
新增标签名称（限8字）、关联分类（可多选，默认全部）、排序权重（数字越大越靠前）
编辑/删除编辑标签名、关联分类、排序；删除需确认
启用/停用停用的标签在C端发布页不显示，已关联的帖子保留标签不变
当前标签库（原型已有20个）
本科、专科、纯租金、倒扣抽点、设备齐全、含水电、含设备、人流量大、包吃住、低价转让、面食、饮品、快餐、小吃、管理岗、技术岗、品牌加盟、沿街旺铺、商业街、中央厨房
4.3 分类管理 P2
功能规则
发布分类管理增删改发布类目（当前10个：校园食堂、校园商圈、医院餐厅、档口转让、企业园区、人才招聘、校外旺铺、生意转让、闲置物品、品牌信息）
二级分类管理为每个一级分类配置二级子类（如校园食堂→本科/专科），支持动态增减
闲置分类管理增删改闲置物品类目（当前4个：餐饮设备、物料耗材、经营资源、其他闲置）
资讯分类管理增删改资讯分类（当前2个：行业政策、创业干货）
启用/停用停用的分类在C端不展示，已关联信息不删除

5.2 退款管理
项目规则
退款发起仅超级管理员可操作，在订单详情中点击【申请退款】
退款金额全额退款（默认）；支持输入部分退款金额
退款原因必填，如"用户投诉""误操作""平台补偿"
退款方式原路退回（微信支付→退回微信零钱；余额支付→退回余额账户；积分抵扣→退回积分）
退款审批可选：退款操作需另一位超管审批（双人确认），可在系统配置中开关
退款效果置顶推广退款→取消置顶，恢复普通排序；手机号解锁退款→收回查看权限
5.3 对账报表 P1
功能规则

导出Excel支持按筛选条件导出订单明细Excel（含全部字段）
微信对账对接微信支付账单下载API，可导入微信账单与平台订单自动对账
